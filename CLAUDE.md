# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

Landing page for **Ipê - Educação Ambiental**, a Brazilian environmental-education company. Single-page site (Portuguese content) built from a client-supplied PDF layout (`site-ipe-layout.pdf`) — use it as the visual reference when adjusting sections.

## Commands

Package manager is **pnpm**.

- `pnpm dev` — dev server on port 3000 (configured in `package.json`)
- `pnpm build` — production build (Nitro server output to `dist/`)
- `pnpm preview` — preview built site
- `pnpm test` — run Vitest suite
- `pnpm test -- <pattern>` — run a single test file/name
- `pnpm check` — Biome lint + format check (use this before committing)
- `pnpm format` / `pnpm lint` — Biome format / lint only

Biome formatter uses **tabs** and **double quotes** (`biome.json`). `src/routeTree.gen.ts` and `src/styles.css` are excluded from Biome.

## Architecture

**Stack:** TanStack Start (SSR meta-framework on TanStack Router) + React 19 + Vite 8 + Nitro (server adapter) + Tailwind CSS v4 + Biome.

**Routing.** File-based via `@tanstack/router-plugin`. Routes live in `src/routes/`; `routeTree.gen.ts` is autogenerated — never edit it. The root shell (HTML document, fonts, devtools) is `src/routes/__root.tsx`. The landing page is a single route at `src/routes/index.tsx` that just composes section components in order.

**Path aliases.** `tsconfig.json` defines both `#/*` and `@/*` → `src/*`. The codebase uses `#/...` (matches `package.json`'s `imports` field). Prefer `#/components/...` over relative paths.

**Component layout.**
- `src/components/sections/` — one file per page section (`Header`, `Hero`, `Services`, `AboutUs`, `MissionVision`, `FinalCta`, `Footer`). Section names are in English even though copy is in Portuguese.
- `src/components/ui/` — shared building blocks: `Logo`, `CtaButton`, `PinkIpeTreeIcon`, `SectionTitle`, `ClippedPhoto`.

**Styling.** Tailwind v4 via `@tailwindcss/vite`. The design system is defined in `src/styles.css` inside an `@theme {}` block — custom colors (`--color-ipe-green-*`, `--color-ipe-pink`, `--color-ipe-pink-dark`, `--color-ipe-brand: #2d8700`) and fonts (`--font-script: Dancing Script`, `--font-sans: Montserrat`) are exposed automatically as Tailwind utilities (e.g. `bg-ipe-brand`, `font-script`). When adding brand colors, add them to `@theme` rather than hardcoding hex in JSX. Google Fonts are loaded via `<link>` in `__root.tsx`. Decorative CSS-only backgrounds (`hero-forest`, `treeline`, `photo-placeholder`, `title-ribbon`) also live in `styles.css`.

**`ClippedPhoto` component.** Renders the diagonal-cut quadrilateral shape used throughout for photos. Implementation detail worth knowing: it uses an inline **SVG `mask-image`** (path with arc commands at each corner) rather than `clip-path: polygon(...)`, because raw `clip-path` produces sharp corners and lets content overflow. The shape's `drop-shadow` is applied via `filter` on an outer wrapper so the shadow follows the masked silhouette. The component accepts `src`/`alt` for future image swap-in; without `src` it renders a green placeholder.

**Server features available but not yet used.** TanStack Start server functions (`createServerFn`) and file-based API routes (`server.handlers` on a route) are configured and ready — no backend code exists yet on this branch.
